## Pasos a implementar por cada sección

1. Crear archivos Theory con el formato indicado, máx 80 columnas por código svg, todas dentro de una subcarpeta "Theory".

2. Por cada sección 01, 02, 03, etc debe haber un "PITFALLS" que recopile todos los errores comunes.

3. Por cada sección debe haber una guia de referencia visual "REFERENCE_SHAPES"

4. Por cada sección crear una carpeta llamada "CHALLENGE" donde habrá de 5 a 10 ejercicios ("challenge_01.md" etc con el mismo formato de los Theory) o más donde el estudiante deberá crear un SVG a medida según una tarea dada.
Si los ejercicios son muy complejos debe de haber algún codigo base para trabajar, sólo enfocarse en los temas de aprendizaje.

5. Glosario visual (GLOSARIO.md) con 2-3 ejemplos de cada implementación con el formato de Theory.

6. Un archivo NOTES.md que contenga notas de compatibilidad, temas que se pueden revisar, recomendaciones extra, etc. Todo lo que no se incluyó en el resto de archivos va aquí.
